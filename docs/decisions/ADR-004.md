# ADR-004: Ring Buffer Event Queue

**Status:** Accepted
**Date:** 2025-01-01

## Context

Events parsed from incoming CDP messages need to be queued until the
caller consumes them via `cdp_next_event()`. The queue must handle
bursty input and overflow gracefully.

## Decision

Use a fixed-size ring buffer (circular buffer) for the event queue.
Default capacity is 64 events, configurable at creation time. When the
queue is full, a `CDP_EVENT_QUEUE_OVERFLOW` event is generated.

## Consequences

- **Positive:** O(1) push and pop
- **Positive:** No dynamic allocation after init
- **Positive:** Bounded memory usage
- **Negative:** Events are lost when the queue overflows
- **Negative:** Queue size must be chosen at creation time
