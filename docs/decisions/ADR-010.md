# ADR-010: Output Buffer with Linear Append

**Status:** Accepted
**Date:** 2025-01-01

## Context

Commands produce JSON that must be sent over the transport. The library
needs a buffer to accumulate outgoing messages until the caller
retrieves them.

## Decision

Use a linear output buffer (1 MiB default). Commands are appended as
newline-delimited JSON. `cdp_get_output()` copies the buffer contents
to the caller's buffer and resets the internal buffer.

## Consequences

- **Positive:** Simple append-only model
- **Positive:** Newline-delimited JSON is easy to parse
- **Positive:** Buffer is consumed atomically
- **Negative:** Fixed buffer size may overflow with many commands
- **Negative:** Caller must retrieve output before buffer fills
