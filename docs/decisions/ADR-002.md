# ADR-002: Callback-Free API

**Status:** Accepted
**Date:** 2025-01-15

## Context

Callback-based APIs are common in event-driven C libraries. However,
they introduce complexity: callback lifetime management, reentrancy
issues, and harder debugging.

## Decision

Use a pull-based model. The caller polls for events via `cdp_next_event()`
rather than registering callbacks. Commands produce output that the
caller explicitly retrieves via `cdp_get_output()`.

## Consequences

- **Positive:** Simpler mental model, no callback lifetime issues
- **Positive:** Easier to debug (caller controls the loop)
- **Positive:** No reentrancy concerns
- **Negative:** Caller must implement their own event loop
- **Negative:** May be less efficient for high-throughput scenarios
