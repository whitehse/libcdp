# ADR-003: JSON Scanning, Not Parsing

**Status:** Accepted
**Date:** 2025-01-15

## Context

CDP messages are JSON. A full JSON parser (like cJSON or Jansson) would
add dependencies and complexity. However, we need to extract specific
fields from incoming messages.

## Decision

Implement a minimal JSON scanner that searches for top-level keys using
string matching. The scanner handles nested braces/brackets to extract
sub-objects as raw strings, but does not build a JSON DOM.

## Consequences

- **Positive:** No external dependencies, small code footprint
- **Positive:** Fast for simple messages
- **Positive:** No heap allocation for JSON DOM
- **Negative:** Cannot handle arbitrarily complex JSON structures
- **Negative:** Escaped quotes in key names may not work correctly
- **Negative:** No JSON validation
