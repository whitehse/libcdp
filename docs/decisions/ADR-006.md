# ADR-006: Caller-Managed Transport (Plumbing Model)

**Status:** Accepted
**Date:** 2025-01-01

## Context

CDP operates over WebSocket. The library could embed a WebSocket client,
or it could leave transport to the caller.

## Decision

The library is transport-agnostic. The caller is responsible for:
- Connecting to the browser's WebSocket endpoint
- Sending the JSON produced by `cdp_get_output()`
- Receiving JSON from the browser and passing it to `cdp_feed_input()`

This is the "plumbing" model — the library handles the protocol layer,
the caller handles the transport layer.

## Consequences

- **Positive:** No WebSocket dependency
- **Positive:** Works with any transport (WebSocket, pipes, files, etc.)
- **Positive:** Easier to test (no mocking needed)
- **Positive:** Smaller library footprint
- **Negative:** Caller must implement transport
- **Negative:** More code for the caller to write
