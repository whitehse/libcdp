# ADR-001: Syscall-Free Design

**Status:** Accepted
**Date:** 2025-01-15

## Context

libcdp is designed to be a portable, embeddable C library. Dependencies
on system calls would limit portability and make testing harder.

## Decision

The library will not make any direct system calls. All I/O is delegated
to the caller. The library operates purely on memory buffers.

## Consequences

- **Positive:** Highly portable, easy to test, no OS dependencies
- **Positive:** Can be used in embedded systems, sandboxes, or restricted environments
- **Negative:** Caller must handle all I/O complexity
- **Negative:** No built-in WebSocket support
