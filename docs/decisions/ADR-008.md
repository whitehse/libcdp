# ADR-008: Auto-Incrementing Command IDs

**Status:** Accepted
**Date:** 2025-01-01

## Context

CDP commands require an "id" field that the browser echoes back in the
response. The caller could manage IDs manually, or the library could
auto-assign them.

## Decision

By default, auto-increment command IDs starting from 1. This can be
disabled via `config.auto_id = 0`, in which case all commands use
the same ID (the current value of next_id, which the caller can set).

## Consequences

- **Positive:** Convenient for most use cases
- **Positive:** No ID collision risk
- **Positive:** Easy to correlate responses
- **Negative:** Auto-increment assumes sequential command processing
- **Negative:** ID wraps around after UINT32_MAX (not a practical concern)
