# ADR-009: Session ID Injection

**Status:** Accepted
**Date:** 2025-01-01

## Context

When attaching to targets via the Target domain, each session has a
unique session ID. Commands must include a "sessionId" field to route
to the correct target.

## Decision

Provide `cdp_set_session_id()` to set the session ID. When set, all
subsequent commands automatically include the sessionId in the JSON
output. The session ID is embedded in the output buffer alongside the
command.

## Consequences

- **Positive:** Transparent session management
- **Positive:** No manual JSON manipulation needed
- **Positive:** Easy to switch between targets
- **Negative:** Session ID changes affect all subsequent commands
