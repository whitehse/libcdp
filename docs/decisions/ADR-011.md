# ADR-011: No Thread Safety

**Status:** Accepted
**Date:** 2025-01-01

## Context

The library could be made thread-safe with mutexes around shared state,
or it could be single-threaded by design.

## Decision

The library is not thread-safe. A single `cdp_ctx_t` must be accessed
from only one thread at a time. The caller is responsible for
synchronization if needed.

## Consequences

- **Positive:** No mutex overhead
- **Positive:** Simpler implementation
- **Positive:** Caller can use their own synchronization strategy
- **Negative:** Not safe to share a context between threads
- **Negative:** Caller must handle thread safety if needed
